1. دورك في المشروع

دورك إنك حولت Laravel API لتجربة ويب واضحة: صفحات عامة، dashboard، إنشاء محتوى، شاشة مراجعة topics، viewer، payments، profile، admin dashboard، وربط كل ده بالـ API والـ local state.

What to say in defense: "أنا مسؤول عن React web app وتجربة المستخدم الأساسية. عملت routing وحماية صفحات، ربطت API، وعملت flow إنشاء المحتوى من اختيار blueprint لحد حفظ وفتح المحتوى."

2. فكرة الويب في NOVAIS

الويب هو الواجهة الأساسية للمستخدم والأدمن. المستخدم يختار نوع المنتج التعليمي، يدخل البيانات، يتابع generation، يراجع outline، يحفظ المحتوى، ثم يفتح الدروس أو المستند. الأدمن من نفس الويب يغير الإعدادات والـ blueprints والخطط والمدفوعات.

3. معمارية الفرونت

App Shell

src/App.js يحدد Router، Layouts، interceptors، والثيم العام.

Pages

كل flow له page: create, generating, topics, course, payment, admin pages.

Components

Sidebar, ExportModal, NotesSidebar, ChatBot, dropdowns, shared UI.

Services

FingerprintService يعمل device id ويتضاف في headers.

5. Routing

src/App.js فيه تقسيم واضح: public layout للـ landing/auth/pages، user layout للـ dashboard/create/profile/payment، admin layout للوحة الإدارة، وstandalone routes للـ course viewer والquiz/certificate/share.

Doctor may ask: ليه HashRouter في Electron؟ عشان production build بيتفتح من file system، والـ BrowserRouter ممكن يكسر refresh/navigation.

6. State Management

  • Auth token والrole في localStorage.
  • Theme darkMode وtheme mode من platform settings.
  • Generation Draft draft بيتحفظ لو generation فشل أو المستخدم رجع.
  • Course Data outline/course id بتتنقل بين generating/topics/course.
  • Language Accept-Language header من اختيار المستخدم.

7. API Integration

Axios interceptor يضيف JWT لو موجود، وdevice id، وAccept-Language. لو حصل 401 يتم تنظيف localStorage والرجوع للـ login. ده يخلي حماية الواجهات متسقة مع الباك.

FrontendBackend endpoint
CreateGET /platform-settings, GET /content-blueprints, POST /generate-course
TopicsPOST /courses, POST /generate-course للadjust
ViewerGET /courses/{id}, POST /generate-lesson, export/share endpoints
Admin/admin/platform-settings, /admin/content-blueprints, /admin/offline-payments

8. Create Flow

  1. src/pages/create.js يجيب blueprints والإعدادات.
  2. يعرض cards/schema حسب نوع المحتوى.
  3. يتحقق من required fields وplan limitations.
  4. يرسل state إلى generating.js.
  5. generating.js يبعت request للباك ويتعامل مع failure/back.
  6. topics.js يعرض outline للمراجعة ثم يحفظ course.
نقطة قوة: الواجهة مش hardcoded على course فقط؛ هي بتقرأ content blueprints وبتعرض fields ديناميكية.

9. Course/Document Viewer

src/pages/course.js يعرض المحتوى، يولد الدروس lazy، يفتح export modal، يشارك الرابط، ويستخدم display terms حسب blueprint عشان العناوين متبقاش كلها "course" في كل منتج.

حدود صريحة: لسه ممكن تلاقي صفحات أو نصوص قديمة فيها terminology عامة للكورسات. PR الحالي في حالة draft وفيه تحسينات ongoing.

10. Admin UI

Platform Settings

src/pages/admin/platformsettings.js لإعدادات المنصة، branding، payments، exports.

Content Blueprints

src/pages/admin/contentblueprints.js لإدارة أنواع المحتوى والـ forms والتعليمات.

Users/CMS

users, paid users, admins, blogs, pages, contacts, notifications.

11. UX وResponsive

الواجهة فيها dashboard، sidebar، صفحات public، صفحات admin، loading states، error handling، export modal، dark mode، ودعم RTL/لغة عبر headers/locales. الهدف إن المستخدم يمشي flow كامل من غير ما يفهم تفاصيل الباك.

12. التعامل مع الأخطاء

  • 401 ينظف token ويرجع login.
  • فشل generation يرجع المستخدم للـ create ومعاه draft بدل ما يضيع البيانات.
  • صفحات payment فيها success/failed/pending.
  • admin layout يمنع غير الأدمن من الدخول.

13. Build/Test Evidence

التحقق الأساسي للفرونت هو CI=true npm run build، ومذكور في readiness docs إنه عدى. Electron build كمان مرتبط بالويب لأنه يبني نفس React bundle. يوجد تحذير npm moderate advisories في packages قديمة مثل react-scripts/quill، لكنه لا يمنع build حسب الفحص.

15. What To Say In Defense

"أنا عملت واجهة ويب API-driven. الواجهة بتقرأ إعدادات وأنواع المحتوى من الباك، تعرض forms ديناميكية، وتحافظ على draft وتجربة المستخدم في حالة الأخطاء. نفس الويب كمان هو أساس نسخة الديسكتوب عن طريق Electron."

Doctor May Ask

1. فين main entry للفرونت؟
src/App.js لأنه بيحدد routes والlayouts والinterceptors.
2. إزاي بتحمي صفحات المستخدم؟
بالتحقق من token/role في الواجهة، والباك برضه يحمي endpoints.
3. هل حماية الفرونت كفاية؟
لا. هي UX فقط. الأمان الحقيقي في backend middleware.
4. ليه استخدمت Axios interceptor؟
عشان كل request ياخد token/language/device id بشكل موحد.
5. إيه X-Device-ID؟
معرف جهاز يتولد في المتصفح ويروح للباك لدعم device-aware flows.
6. إزاي create page ديناميكية؟
بتقرأ /content-blueprints وتبني fields حسب schema.
7. ليه في topics page قبل الحفظ؟
عشان المستخدم يراجع outline قبل ما يتحفظ كcourse/document دائم.
8. لو generation فشل؟
نرجع للcreate ومعانا draft بدل فقدان المدخلات.
9. إيه الفرق بين create وgenerating؟
create يجمع البيانات، generating يتعامل مع request/loading/error.
10. إزاي تعرض أنواع محتوى غير course؟
باستخدام display terms القادمة من blueprint والmetadata المحفوظة.
11. فين export؟
ExportModal.js يطلب PDF/PPT blob من backend ويحمله.
12. إزاي share بيشتغل؟
الفرونت يطلب share link من الباك ويعرض رابط public route.
13. ليه Electron يستخدم نفس الويب؟
عشان نقلل duplication ونخلي الديسكتوب يستفيد من نفس React UI.
14. ليه HashRouter للديسكتوب؟
لأن file protocol لا يدعم server-side fallback زي dev server.
15. إزاي بتتعامل مع dark mode؟
من platform settings/localStorage وتطبيق class/theme على الواجهة.
16. فين admin routes؟
داخل nested admin layout في App.js.
17. لو غير أدمن فتح admin؟
AdminLayout يتحقق من profile ويرجع المستخدم.
18. إيه platform settings UI؟
واجهة للأدمن لتغيير branding/features/payments/exports بدون تعديل كود.
19. إيه content blueprints UI؟
واجهة للأدمن لإدارة أنواع المنتجات التعليمية وحقولها وتعليماتها.
20. إزاي payment UI مربوط؟
صفحات payment/pricing/success/failed تتكلم مع payment endpoints وتعرض الحالة.
21. هل في legacy نصوص؟
نعم ممكن بقايا "course" في أجزاء قديمة، والتحسينات في PR draft.
22. إيه أهم build command؟
CI=true npm run build.
23. لو API رجع 401؟
interceptor يمسح بيانات auth ويعيد التوجيه للlogin.
24. هل الويب وحده يقرر plan limits؟
لا. الويب يعرض UX، والباك يطبق limit فعليا.
25. إيه أكبر challenge؟
ربط UI واحد بأنواع محتوى متعددة بدون hardcoding زائد.
26. إزاي تشرح contribution بتاعك؟
ربطت المستخدم بالـ AI backend في flow كامل قابل للاستخدام والإدارة.

Common Mistakes

  • ما تقولش إن الفرونت هو مصدر الأمان؛ هو طبقة عرض فقط.
  • ما تنساش تذكر draft recovery في generation flow.
  • ما تخلطش بين public user routes وadmin nested routes.
  • لو اتسألت عن terminology، قول إن فيه تحسينات product-aware ولسه PR draft.
